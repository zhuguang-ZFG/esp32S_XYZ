## 0. 方法论

### 0.1 核对三级制

所有器件统一按三级核查，不跳级：

| 级别 | 内容 | 产物 | 不通过的处理 |
|---|---|---|---|
| **L1 引脚** | 对每个在原理图 / GPIO 文档中出现的引脚，核对 datasheet 的 Pin Description：引脚号、方向（I/O/PI/AI）、内部上/下拉、是否 strapping 等 | 引脚矩阵表 + 冲突列表 | 登记到 §A 风险表，标注 `L1-risk` |
| **L2 配置** | 核对外围配置：上下拉、去耦电容、时钟/复位序列、I2C/SPI/UART 地址与方向、PWM 范围 | 配置差异表 | 登记到 §A，标注 `L2-risk` |
| **L3 应力** | 核对实际工作点是否在 datasheet 的 absolute maximum ratings 与 recommended operating conditions 之内：电流、电压、温度、频率 | 应力清单 | 登记到 §A，标注 `L3-risk` |

硬约束：**任何一级不通过但软件在用**的项目，在 §A 登记前**不得标"通过"**。

## 1. 主控芯片

### 1.1 ESP32-S3-WROOM-1-N16R8（U1 / U8 双片）

数据源：`[D1]` ESP32-S3 Series Datasheet v2.2，章节 §3 *Boot Configurations*、§3.1~§3.4 *Strapping Pins / Boot Mode / VDD_SPI Voltage / JTAG Signal Source Control*。

#### 1.1.1 strapping pin 总览（datasheet §3.1）

ESP32-S3 共 4 根 strapping pin：`IO0 / IO3 / IO45 / IO46`，复位瞬间被采样决定启动模式 / VDD_SPI 电压 / JTAG 路径。datasheet 关键约束摘录：

| Strapping pin | 作用 | datasheet 关键句 |
|---|---|---|
| `IO0` | Boot mode（SPI Boot / Download Boot） | "weak internal pull-up at reset"，外部默认拉高=SPI Boot，按下到 GND=Download Boot |
| `IO3` | JTAG Signal Source Control | "**This pin does not have any internal pull resistors** and the strapping value must be controlled by the external circuit that **cannot be in a high impedance state**" |
| `IO45` | VDD_SPI 电压选择（3.3 V vs 1.8 V） | 默认弱下拉=3.3 V（适配 N16R8 内置 flash） |
| `IO46` | ROM messages 打印控制（与 IO0 联合参与 boot mode） | "weak internal pull-down at reset" |

本板**两颗 ESP32-S3 都对这 4 根 strapping pin 有占用**，逐根核对见下文。

#### 1.1.2 IO0 — BOOT 按键

| 项 | U1 | U8 |
|---|---|---|
| 网络名 | `BOOT` `[D2]` | `AI_BOOT` `[D2]` |
| 用途 | 下载启动按钮（接到 GND 时进 download mode） | 同 U1 |
| datasheet 期望 | 弱内部上拉 + 外部按键到 GND；空闲时为高 | 同 |
| 现状判定 | **符合**。IO0 仅作为 Type-C 自动下载电路与手动按键输入，复位时无外部驱动器，靠内部上拉保持高电平 |
| 风险 | 无（合规用法） | 无 |

#### 1.1.3 IO3 — JTAG Signal Source Control（关键风险）

datasheet §3.4 原文（页 26）：

> The strapping pin GPIO3 can be used to control the source of JTAG signals during the early boot process. **This pin does not have any internal pull resistors** and the strapping value must be controlled by the external circuit that **cannot be in a high impedance state**.

现状：

| 项 | U1 | U8 |
|---|---|---|
| 网络名 | `TMC2208-1_DIR` `[D3]` | `IO3_DVP_VSYNC` `[D3]` |
| 方向 | MCU → 外部（OUTPUT） | 外部 → MCU（INPUT / 摄像头 VSYNC） |
| 复位瞬间驱动源 | HR4988E DIR 输入端（高阻） | 摄像头模块的 VSYNC 输出端 |
| 复位状态 | **外部高阻 → 违反 datasheet "cannot be in a high impedance state"** | 取决于摄像头上电时序。多数 DVP 模组在主控释放前 PWDN=1，VSYNC 输出关；仍需验证 |

**U1 侧风险等级：L2**（见 §A.1 `R-001`）：

- 表现：复位瞬间 IO3 悬空，JTAG signal source 不确定 → 可能偶发 JTAG 状态异常、烧录不稳定、量产烧写良率下降
- 非 L1：不会立即烧毁硅片，复位过后 IO3 进 GPIO 模式后软件可驱动
- 缓解选项（任选一）：
  1. PCB 改版：IO3 到 GND 加 10 kΩ 下拉（最低成本；与 `TMC2208-1_DIR` / HR4988E DIR 信号方向兼容——固件上电时默认输出低，等效等待明确方向指令时为低电平）
  2. PCB 改版：把 `TMC2208-1_DIR` 从 IO3 迁到非 strapping 的未用 GPIO（U1 空闲 IO1 / IO2 / IO19 / IO20 可选）
  3. 软件侧：在 efuse 烧 `EFUSE_DIS_PAD_JTAG` 永久关 JTAG（代价：永远失去 JTAG 调试能力，v2 §24 自检体系将无法通过硬件探针查步进时序）
- **2026-05-14 轮 2 修订**：PADS `.txt` 的 `*SIGNAL*` 显示 `TMC2208-1_DIR` 在主控页连接 `U1.15`，在驱动页连接 `R4.2`；`R4.1` 再串到 `U52.19 DIR`。未发现该网络接入 DGND 或电源的上下拉器件，因此设计源文件层面判定 **IO3 无外部下拉**。PCB PDF 为图形交付件，不能替代 netlist 证明隐藏下拉；R-001 保持 open。

**U8 侧风险等级：L3**（见 §A.1 `R-002`）：

- 表现：DVP VSYNC 是摄像头输出，取决于上电时序是否让 VSYNC 在主控释放复位前已稳定到确定电平
- 需要摄像头 datasheet 补证：上电时 VSYNC 是否立即拉低或受 PWDN/RESET 控制
- 轮 2 待摄像头型号确认后回填

#### 1.1.4 IO45 — VDD_SPI 电压选择

datasheet §3.3：复位时采样 IO45，**默认弱内部下拉**（=0），选择 VDD_SPI = 3.3 V。N16R8 模组内置的 flash/PSRAM 标称工作电压 3.3 V，因此 IO45 在复位瞬间**必须保持低**才能正常上电。

现状：

| 项 | U1 | U8 |
|---|---|---|
| 网络名 | `LASER_CONTROL` `[D3]` | `IO45_I2S_DO` `[D3]` |
| 方向 | MCU → MOS 栅极（OUTPUT） | MCU → ES8311 SDIN（OUTPUT） |
| 复位瞬间外部驱动 | 无（IRFZ44N 栅极是高阻 MOS，需 MCU 驱动） | 无（ES8311 SDIN 是输入，高阻） |
| 复位时电平 | 内部弱下拉=0 → IRFZ44N 栅极=低 → 激光 MOS 关断 → ✓ 安全 | 内部弱下拉=0 → ES8311 SDIN=0 → 启动期没有音频数据 → ✓ 无害 |
| 复位时 VDD_SPI strapping 值 | 0 → VDD_SPI=3.3 V ✓（与 N16R8 内置 flash 一致） | 同 |

**U1 / U8 侧风险等级：L3**（见 §A.1 `R-003`）：

- 复位瞬间内部弱下拉能保住正确 strapping 值，但**任何外部上拉**（哪怕 100 kΩ）都会改写 strapping 到 1.8 V，导致 N16R8 上不了电
- 当前原理图未识别出 IO45 上有外部上拉，待轮 2 PCB 走线核对确认
- 软件层面：U1 在 `LASER_CONTROL` 上电默认必须输出 0（激光关），既符合 strapping 默认也符合 §10bis.8 安全要求；U8 在 I2S 初始化前不要把 IO45 配成输出高
- **附加风险**：`LASER_CONTROL` 用 strapping pin 在量产时如果不慎接反外部电路（例如试产里挂了上拉电阻试调），会让设备直接砖；建议未来 PCB 迭代把它迁离 strapping pin

#### 1.1.5 IO46 — ROM messages / 联合 boot mode

datasheet §3.1：IO46 与 IO0 联合决定 boot mode 与 ROM 打印；**默认弱内部下拉**。

现状：

| 项 | U1 | U8 |
|---|---|---|
| 网络名 | `TMC2208-1_STEP` `[D3]` | `IO46_DVP_HREF` `[D3]` |
| 方向 | MCU → HR4988E STEP（OUTPUT） | 摄像头 → MCU（INPUT） |
| 复位瞬间外部驱动 | 无（HR4988E STEP 是输入端，高阻） | 摄像头 HREF 输出端 |
| 复位时电平 | 内部弱下拉=0 → ✓ 等价默认 boot mode | 取决于摄像头模组 PWDN/RESET 时序 |

**U1 侧风险等级：L3**（见 §A.1 `R-004`）：

- 内部弱下拉能保正确 strapping
- 风险点：若 IO3 改版后通过外部上拉强制为高，IO46 仍保持下拉，则 boot mode 落到合法组合；但 PCB 出现意外上拉就会改 boot mode
- 软件层面：U1 上电默认 STEP=0，启动期不会主动驱动；运行期固定为脉冲驱动，不影响 strapping（strapping 仅在复位瞬间生效）
- 与 IO45 同样建议未来 PCB 迭代迁离 strapping pin

**U8 侧风险等级：L3**（见 §A.1 `R-005`）：

- 与 IO3 同源问题：DVP HREF 是摄像头输出，复位瞬间是否高阻取决于摄像头时序
- 多数 OV/GC 系列 DVP 模组在 PWDN=1 时所有数据/同步线被驱动到低或高阻，需 datasheet 二次核实
- 轮 2 摄像头型号确认后回填

#### 1.1.7 §1.1 小结

本轮在 ESP32-S3 主控芯片上发现并登记的风险条目：

| 编号 | 位置 | 等级 | 摘要 |
|---|---|---|---|
| `R-001` | U1.IO3 = `TMC2208-1_DIR` 网络 / HR4988E DIR | L2 | strapping pin 复位瞬间高阻，违反 datasheet §3.4 "cannot be in a high impedance state" |
| `R-002` | U8.IO3 = DVP_VSYNC | L3 | strapping pin 由摄像头驱动，需摄像头 datasheet 二次核对上电时序 |
| `R-003` | U1/U8.IO45 | L3 | strapping pin（VDD_SPI 选 3.3 V）依赖内部弱下拉；任何外部上拉会砖板，需 PCB 走线核对 |
| `R-004` | U1.IO46 = `TMC2208-1_STEP` 网络 / HR4988E STEP | L3 | strapping pin 同 R-003 性质，依赖内部弱下拉 |
| `R-005` | U8.IO46 = DVP_HREF | L3 | strapping pin 由摄像头驱动，与 R-002 同源 |
| `R-006` | U8.IO38/39/40 | 已关闭 | PADS `.txt` 已证明 U8.31/32/33 分别连接 IO38/39/40 |
| `R-007` | HR4988E ROSC | L2 | `R13/R29/R42/R55=1K` 使 `tOFF≈1.2us`，偏离 datasheet 约 30us 推荐用法 |
| `R-008` | HR4988E CP1/CP2 | L2 | `C7/C14/C23/C30=22nF`，低于 datasheet 推荐 `0.1uF/50V` |

合规结论：

- IO0 / IO45 / IO46 在合规默认电平下都能保住 strapping，但 **IO45 / IO46 都用 strapping pin 接外设是脆弱设计**：未来 PCB 任何附加上下拉就会改 strapping
- IO3 是当前**唯一明确不合规**的项（datasheet 原文反对），需上板前给出处理方案
- U8.IO38/39/40 的固件配置与 PADS 源文件一致；IO35/36/37/41/42 仍不得作为固件扩展脚使用，除非后续补齐 datasheet 与实物证据

##### 1.2.1 L1 引脚与固件接口

| 项 | datasheet 要求 | 本板连接 | 判定 |

| `STEP` | 上升沿有效；高/低最小脉宽均 `1 us` | U1 STEP 经 `1K` 串联到 HR4988E；固件 `DEFAULT_STEP_PULSE_MICROSECONDS=4` | 通过 |
| `DIR` | STEP 有效沿前建立 `200 ns`，后保持 `200 ns` | U1 DIR 经 `1K` 串联到 HR4988E；本轮固件新增 `STEP_PULSE_DELAY=1 us` | 通过 |
| `ENABLE#` | 低有效；高电平关断输出 | `MOTOR_EN` 经 `1K` 串联，另有 `9.1K` 拉到 `+3V3` | 通过：复位默认关断，`DEFAULT_INVERT_ST_ENABLE=0` 与低有效匹配 |
| `RESET# / SLEEP#` | 低有效；SLEEP 拉高后需等待约 `1 ms` 才能 STEP | 与 `MS1/MS2/MS3` 同网，经 `1K` 拉到 `+3V3` | 通过：不进入 sleep，固件无需 wake 延迟 |
| `MS1/MS2/MS3` | `111` = `1/16 Step` | 三脚同网并拉高 | 通过：固定 1/16 微步，固件 steps/mm 需按 1/16 标定 |
| `REF` / `SENSE` | `ITripMAX = VREF / (8 * Rs)`；`VSENSE` 不得超过 `0.5 V` | `R14=9.1K` 到 3V3、`R15=2.2K` 到 GND，估算 `VREF≈0.64 V`；`Rs=0.1Ω` | 通过但需实测：估算限流约 `0.80 A/相` |

##### 1.2.2 L2 配置差异

- **ROSC 配置存在风险**：以 U52 为例，`U52.13 ROSC -> R13.2`，`R13.1 -> DGND`，`R13=1K`。datasheet 给出 `tOFF≈ROSC/825 us`，因此 1K 约等于 `1.2 us`，接近 datasheet 的 blanking 典型值 `1 us`，明显偏离 datasheet 推荐示例（ROSC 接 VDD/GND 或 25K 时约 `30 us`）。同类通道还包括 `R29/R42/R55`。登记 `R-007`。
- **CP1/CP2 飞跨电容偏小**：datasheet 推荐 CP 电容 `0.1uF/50V`；本板 U52 为 `C7=22nF`，其余通道对应 `C14/C23/C30=22nF`。VCP 储能电容 `C8/C15/C24/C33=0.1uF` 符合推荐。登记 `R-008`。
- **VREG 电容偏大但方向安全**：datasheet 推荐 `0.22uF/16V` 到 GND；本板 `C6/C13/C19/C29=2.2uF` 到 GND。该项先标 L3，待实测确认启动/温升，无需立即改固件。

##### 1.2.3 L3 应力与结论

- 功率电源 `12V` 与逻辑电源 `3V3` 均在 HR4988E 推荐工作区间。
- 估算 `VREF = 3.3 * 2.2 / (9.1 + 2.2) ≈ 0.64 V`，`ITripMAX = 0.64 / (8 * 0.1) ≈ 0.80 A`；对应 `VSENSE≈0.08 V`，低于 `0.5 V` 绝对上限。真实电流仍需用 VREF 实测值与电机温升确认。
- 固件本轮新增 `STEP_PULSE_DELAY=1 us`，使 DIR/MSx 到 STEP 上升沿的建立时间有明确余量；新板或已擦 NVS 设备使用默认值即可。若沿用旧 NVS，需确认 `$Stepper/Direction/Delay` 不为 0。
- 本轮不改 GPIO 分配；需要硬件侧继续处理的是 `R-007/R-008` 两个外围配置项。

#### 1.6.3 §1.6 小结

- HX711 的 L1 pin map 与 U1 GPIO 基准一致：`SCLK=GPIO7`，`DOUT=GPIO15`。

PADS 符号中 `LM1117` 引脚为 `3=VIN`、`2=VOUT`、`1=ADJ/GND`、`4=VOUT/tab` `[D3]`。本板连接如下：

字段含义：

- **状态 `open`**：风险已确认，需在上板前或上板时处理
- **状态 `pending-evidence`**：风险方向已识别，需要补 datasheet / PCB 证据后定级
- **状态 `pending-verification`**：仅有疑问，未达到风险确认门槛
- **状态 `open-with-mitigation`**：已有硬件/软件缓解证据，但关键 datasheet 或实测证据未闭环

- **状态 `accepted`**：知晓但项目决定接受
- **状态 `accepted-source`**：设计源文件已证明配置，生产前仍需抽测实物一致性
