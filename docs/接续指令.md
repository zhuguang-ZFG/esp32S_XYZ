### 已完成的调研

**ESP32-S3-WROOM-1-N16R8**：

- datasheet URL：<https://documentation.espressif.com/esp32-s3_datasheet_en.pdf>（87 页，v2.2）
- 本地缓存：`C:\Users\zhugu\.claude\projects\C--Users-zhugu-Desktop-xue-esp32S-XYZ\ded71cde-b3ee-41b1-bb94-34646a6d1189\tool-results\webfetch-1778700227013-kefhlj.pdf`
- 关键章节：§3 Boot Configurations、§3.1~§3.4 strapping/boot/VDD_SPI/JTAG
- **重大发现 #1**：IO3 是 strapping pin（JTAG Signal Source Control），datasheet 原文：
  > The strapping pin GPIO3 can be used to control the source of JTAG signals during the early boot process. **This pin does not have any internal pull resistors** and the strapping value must be controlled by the external circuit that **cannot be in a high impedance state**.
- **对照现状**：U1 机型文件 `dlc_motor_control_p1.h` 将 IO3 用作 `TMC2208-1_DIR`（IC 输入端，复位瞬间等效高阻）
- **风险级别**：L2，上板可能表现为偶发 JTAG 异常 / 烧录不稳定
- **缓解选项**：① 加 10kΩ 外部下拉到 GND ② 把 `TMC2208-1_DIR` 换到非 strapping GPIO（需改 PCB）③ 确认 eFuse 已烧 `EFUSE_DIS_PAD_JTAG` 强制关 JTAG（但这会阻塞将来调试）
- **待你核实**：PDF PCB 图里 IO3 或 `TMC2208-1_DIR` 附近是否已经有外部上/下拉电阻

其他 strapping 事实：

- IO0 / IO46：boot mode 控制，现状 IO0 = BOOT 按键、IO46 = `IO46_DVP_HREF`（U8）或 `TMC2208-1_STEP`（U1）
- IO45：VDD_SPI 电压控制，现状 = `LASER_CONTROL`（U1）或 `IO45_I2S_DO`（U8）—— 需核实是否影响 VDD_SPI

所以**至少还有 3 处 strapping pin 冲突需要在报告里登记**：

- U1 IO46 用作 X_STEP（strapping）
- U1 IO45 用作 LASER_CONTROL（strapping）
- U8 IO45 用作 I2S_DO（strapping）
- U8 IO46 用作 DVP_HREF（strapping）

这都是"已经发生的弱证使用"，需要 datasheet 二次核实是否有实际影响。
