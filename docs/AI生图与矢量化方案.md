# AI 生图与矢量化方案

**日期**: 2026-05-16
**状态**: 已实现并集成到 manager-api

## 1. 方案概述

写字机需要将用户的文字描述转换为可执行的笔绘路径。完整 pipeline：

```
用户输入 "画一只猫"
    ↓
qwen-image-2.0 生成 512×512 简笔画 PNG (~2.5s)
    ↓
下载图片 (~0.5s)
    ↓
Zhang-Suen 骨架化 → 中心线提取 (~0.1s)
    ↓
路径追踪 + RDP 简化 (~0.2s)
    ↓
PathOptimizer 贪心 TSP 重排 (<0.01s)
    ↓
SVG path → run_path 下发设备
    ↓
设备固件 M→G0(抬笔移动) L→G1(落笔画线)
```

**总耗时: ~3.5 秒**

## 3. 矢量化算法

### 3.1 为什么不用轮廓追踪

轮廓追踪（marching squares）描的是线条的**外边缘**（两侧轮廓），不是笔画中心线。
写字机需要的是笔画的**中心路径**（一笔画过去的轨迹）。

### 3.2 骨架化 (Skeletonize) 方案

采用 Zhang-Suen 细化算法，将线条区域收缩为 1 像素宽的中心线：

```
原始图像 (黑色线条)     →     骨架 (1px 中心线)
████████                      ────────
████████          →           
████████                      
```

### 3.3 完整算法步骤

1. **灰度化 + 自动裁白边**: 去除图片四周空白区域
2. **缩放到 512×512**: 保持细节的同时控制计算量
3. **二值化**: 阈值 128，黑色为前景
4. **Zhang-Suen 骨架化**: 迭代细化，保留拓扑连通性
5. **交叉点感知路径追踪**:
   - 识别端点（1 邻居）和交叉点（3+ 邻居）
   - 从端点开始追踪，在交叉点处断开（pen up）
   - 保证小特征（眼睛、细节）不被错误合并
6. **路径合并**: 距离 ≤3px 的端点自动连接
7. **RDP 简化**: Douglas-Peucker 算法，epsilon=0.5，保留曲线细节
8. **输出**: SVG path（M/L 命令），最多 800 个点

### 3.4 Java 实现

`BitmapToSvgVectorizer.java` — 纯 Java 实现，无外部依赖：
- Zhang-Suen 细化: ~50 行核心逻辑
- 路径追踪: 基于 HashSet 的 8-连通邻域搜索
- RDP 简化: 递归实现
- 性能: 512×512 图像矢量化 < 0.3s

## 4. 路径优化算法

### 4.1 问题

骨架化产生多个路径段（10~50 段），段间顺序随机。
笔在段间需要抬起移动（空移），未优化时空移距离可能很长。

### 4.2 贪心最近邻 TSP

`PathOptimizer.java` 实现：

```
输入: [段A, 段B, 段C, ...]  (每段有起点和终点)
笔位置 = (0, 0)

循环:
  对所有未使用的段:
    计算 段.起点 到 笔位置 的距离
    计算 段.终点 到 笔位置 的距离 (若更近则反向画)
  选最近的段 → 画它 → 更新笔位置
```

### 4.3 效果

- 空移距离减少 40~60%
- 复杂度 O(n²)，n=段数（通常 10~50），耗时可忽略
- 支持路径反转：如果段的终点比起点更近，自动反向绘制

## 5. 服务端集成

  dashscope:

## 6. 性能总结

| 阶段 | 耗时 | 说明 |
|------|------|------|
| AI 生图 | 2.5s | qwen-image-2.0, 512×512 |
| 图片下载 | 0.5s | OSS 加速链接 |
| 矢量化 | 0.3s | Zhang-Suen + 路径追踪 + RDP |
| 路径优化 | <0.01s | 贪心 TSP |
| **总计** | **~3.5s** | 用户感知延迟 |

## 7. 已知限制与后续优化方向

1. **骨架化在交叉点处仍有少量断线**: 已通过端点合并缓解，但复杂图形仍可能有
2. **RDP epsilon 固定为 0.5**: 可考虑根据图形复杂度自适应调整
3. **未支持曲线命令**: 当前只用 M/L（直线段），未来可加 Q/C（贝塞尔曲线）提升平滑度
4. **多 AI 模型支持**: 可扩展为多模型配置，按速度/质量偏好选择
5. **预置素材库**: 常见图形（星星、心形等）可直接使用预置 SVG，0ms 响应
