# M6.6 Consumables Status

Status: implemented for manager-api backend contract and manager-mobile manual controls.

Evidence Boundary:

This status is local/design-time evidence only. It is not a completion declaration and does not replace real paper sensing, ink sensing, pressure sensing, or real-device mileage accuracy evidence.

Implemented:

- Added `device_supplies` persistence with `paper_slot_state`, `pen_installed_at`, `pen_ink_percent_est`, and `pen_mileage_mm`.
- Added `POST /api/v1/devices/{deviceId}/supplies` for manual consumable state updates by the active bound account.
- manual supply updates select active bindings by `updated_at` desc and binding `id` desc before accepting the account/device update.
- `pen_ink_percent_est` accepts only integer values from 0 to 100; out-of-range values are rejected before insert/update.
- Added `E_NO_PAPER` and a soft paper gate for `write_text`, `draw`, and `draw_generated` submissions.
- The paper gate runs before task persistence and before forwarding accepted tasks to DeviceServer.
- Added manager-mobile device detail controls for marking paper loaded, marking paper empty, and recording a new pen.
- Manager-mobile maps `E_NO_PAPER` to a direct paper-ready prompt.
- `run_path` tasks that newly reach `done` automatically add their line-segment path length to `pen_mileage_mm`.
- `write_text` and `draw_generated` submissions persist the actual dispatched `run_path` capability and params, so completed projected tasks also add mileage from the path that was sent to DeviceServer.
- Added operations runbook `ops/runbooks/m6-consumables-hardware.md` for real-device consumables acceptance, `E_NO_PAPER` drills, mileage checks, sensor integration gates, and evidence capture.
- The runbook mileage drill requires the exact `run_path` payload, an expected mileage calculation using `sqrt(dx^2 + dy^2 + dz^2)` per line segment, and a <= 1 mm tolerance unless firmware/controller rounding evidence justifies a wider tolerance.

Validation:

- Java unit coverage: `DeviceSupplyServiceImplTest`, `AppV2ServiceImplTest`, `AppV2ControllerTest`.
- Static CI coverage: `tests/ci/test_manager_api_consumables.py`, `tests/ci/test_manager_mobile_device_info.py`.
- Static CI guards `tasks.dispatch_capability` / `tasks.dispatch_params_json` migration and the projected-task mileage fallback.
- Static CI guards `pen_ink_percent_est` range validation, including rejection of values below 0 or above 100 before persistence.
- Runbook coverage: `tests/ci/test_manager_api_consumables.py` guards the hardware acceptance drill and sensor limitation language.

Limitations:

- Manager-mobile uses the existing device detail page rather than a dedicated consumables page.
- Pen mileage is an estimate from completed dispatched `run_path` geometry; it does not use a hardware ink or pressure sensor.
- `unknown` paper state is allowed through; only user-marked `empty` blocks writes/draws.
- The runbook is not physical hardware evidence; real paper sensing, ink sensing, pressure sensing, and mileage accuracy still require staging or production device validation.
