# M6.1 隐私政策与权限授权状态

日期：2026-05-15

## Evidence Boundary

This status is local/design-time evidence only. It is not a completion declaration and does not replace WeChat console privacy declarations, real permission prompt screenshots, or WeChat review approval evidence.

## 已完成

- 小程序新增 `pages/settings/privacy-permissions` 隐私与权限页。
- 设置页新增“隐私协议与系统授权”入口。
- 权限页复用现有隐私政策页入口：`/pages/login/privacy-policy-zh`。
- 权限页按用途拆分三类授权：
  - 麦克风：`scope.record`
  - 蓝牙：`uni.openBluetoothAdapter`
  - Wi-Fi：`uni.startWifi`
- 每类权限都有未授权兜底提示，并提供 `uni.openSetting` 跳转系统授权设置。
- 配网页增加蓝牙 / Wi-Fi 权限提示和权限设置入口。

## Build Artifact Coverage

- `manifest.config.ts` declares `mp-weixin.permission.scope.record`, `mp-weixin.permission.scope.userLocation`, and `requiredPrivateInfos: getLocation`.
- `vite.config.ts` patches the generated WeChat `dist/build/mp-weixin/app.json` permission artifact after `build:mp-weixin`.
- Local and CI checks must parse the generated `app.json`; source manifest checks alone are not sufficient.

## 验收口径

- 用户能从设置页进入隐私与权限页。
- 用户能查看隐私政策。
- 麦克风、蓝牙、Wi-Fi 可分别触发授权动作。
- 未授权时有明确兜底提示，并能进入系统授权设置。
- 配网页能引导用户处理蓝牙 / Wi-Fi 授权问题。

## 当前限制

- 当前为小程序侧最小闭环，未接入微信小程序后台隐私接口配置校验。
- 权限状态以本次页面操作结果展示，未做跨端持久化同步。
- 真实微信审核仍需在小程序后台补齐隐私协议声明和 API 权限用途。
