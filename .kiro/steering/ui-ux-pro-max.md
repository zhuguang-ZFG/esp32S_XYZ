# UI/UX Pro Max - 小程序界面规范

**版本**: v1.0
**创建日期**: 2026-05-15
**状态**: 生效中
**基准**: `docs/架构定稿-v2.md §2bis, §8, §8bis`

## §1 核心原则

### 1.1 模板优先（架构定稿-v2 §8bis）

```
✅ 必须基于 GitHub 成熟 uni-app + mp-weixin 模板
✅ 模板选型登记在 docs/ui-template.md
❌ 禁止手搓页面骨架、组件库、主题
❌ 禁止自建 CSS 框架
```

### 1.2 小程序定位：身份/订阅中继（架构定稿-v2 §2bis）

```
✅ 小程序负责：身份建立、订阅登记、用户手动下发任务
❌ 小程序不负责：直连设备、持久化任务数据、承载语音流
❌ 小程序不是数据中继——关了任务照跑
```

## §2 页面结构

### 2.1 最小可用页面（M2.8）

| 页面 | 职责 | 约束 |
|------|------|------|
| 登录页 | 微信 code → JWT | 基于模板的登录页，只改色值和品牌 logo |
| 设备列表页 | 展示已绑定设备 | 列表项：设备名、在线状态、最近任务摘要 |
| 设备详情页 | 设备控制 + 实时状态 | 归零按钮 + 状态卡片 + Edge-A 调试区 |

### 2.2 扩展页面（M3+）

| 页面 | 何时添加 |
|------|----------|
| 写字页 | M3.6 write_text demo 后 |
| 画图页 | M3.9 draw_generated e2e 后 |
| 语音/声纹页 | M4.5 声纹注册后 |
| 设置/配网页 | M5.1 BLE 配网后 |
| 健康检查页 | M5.8 后 |

### 2.3 禁止事项

```
❌ 任何超出"模板原生组件改色"的 UI 工作在 M2.8 阶段做
❌ 手搓 tabbar、导航栏、picker、modal
❌ 自建 loading skeleton
❌ 自建 toast/notification 组件
```

## §3 状态展示规范

### 3.1 设备状态

| 状态 | 视觉呈现 | 颜色 |
|------|----------|------|
| IDLE | 空闲 | 绿色 |
| HOMING | 归零中 | 蓝色闪烁 |
| RUNNING | 执行中 | 蓝色 |
| PAUSED | 暂停 | 黄色 |
| ALARM | 告警 | 橙色 |
| ERROR | 错误 | 红色 |
| ESTOP | 急停 | 红色闪烁 |
| OFFLINE | 离线 | 灰色 |

### 3.2 任务五态

| 状态 | 中文文案 | 图标 |
|------|----------|------|
| accepted | 已接受 | 时钟 |
| running | 执行中 | 旋转圆 |
| done | 完成 | 勾 |
| failed | 失败 | 叉 |
| cancelled | 已取消 | 停止 |

## §4 Edge-A 交互

### 4.1 WSS 连接

```
连接：/ws/v1/client
首帧：{"type":"auth","token":"<JWT>"}
订阅：{"type":"subscribe_device","device_id":"dev_xxx"}
事件：{"type":"event","event_type":"...","seq":123,...}
重连：since_seq 补齐
```

### 4.2 连接状态展示

| 状态 | 文案 |
|------|------|
| connecting | 连接中... |
| connected | 已连接 |
| disconnected | 连接断开，重试中 |
| auth_failed | 登录过期，请重新登录 |

## §5 加载与错误状态

```
每个页面必须覆盖：
□ 加载中（skeleton/spinner）
□ 空状态（"暂无设备" / "绑定你的第一台设备吧"）
□ 错误状态（网络错误 / 设备离线 / 权限不足）
□ 边界状态（权限过期、token 失效、设备已转赠）
```

## §6 验收标准

### M2.8 通过标准

```
□ 三页面可通过微信扫码体验
□ 登录、绑定、查看设备列表、进入设备详情全流程无手动操作
□ 点"归零"能看到状态变化（先 mock，再真实链路）
□ 设备离线时有明确离线提示
□ 网络错误时不崩溃
```
